## Purpose

Ensuring code readability, maintainability, and adherence to team conventions.

## When to Apply

Apply during any code review or development phase to maintain a high bar for code quality.

## Steps

1. Verify names are readable and descriptive (no `tmp`, `data`, `x`).
2. Ensure functions do one thing (SRP) and avoid deep nesting.
3. Check for code duplication (DRY) and redundant logic.
4. Verify adherence to project-specific style guides and patterns.
5. Review JSDoc and internal documentation for clarity.

## Output Format

- Categorized style findings in REVIEW.md.

- Specific refactoring suggestions.
