## Procedure

1. Receive the task brief and requirements for a UI component or view.
2. Define the component hierarchy and state management strategy.
3. Consult `references/design-tokens.md` for consistent styling.
4. Produce a written specification for the developer to follow.

## Gotchas

- Ensure the architecture is strictly scoped to the UI layer.
- Refer to `references/design-tokens.md` for design token definitions.

## Validation loop

The ui-architect ensures that the proposed architecture is modular, scalable, and adheres to the project's design tokens.
