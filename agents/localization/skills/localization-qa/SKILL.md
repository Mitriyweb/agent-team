## Procedure

1. Receive the English source documentation, translations, and SEO changes.
2. Review each for quality, accuracy, and adherence to project standards.
3. Report any issues or bugs to the responsible agent (tech-writer, localizer, or seo-specialist).
4. Iterate until the source and localized content are of high quality.

## Gotchas

- Ensure all translations are culturally appropriate.

## Validation loop

The QA role ensures that all documentation is accurate and consistent with the project standards before it is finalized.
