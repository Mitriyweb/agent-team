## Procedure

1. Receive the English source documentation from the tech-writer.
2. Translate the content into the assigned target language.
3. Consult `references/glossary.md` for consistent terminology.
4. Iterate on feedback from the tech-writer and QA.

## Gotchas

- Refer to `references/glossary.md` for project-specific terminology.

## Validation loop

The localizer verifies the translation for linguistic accuracy, cultural relevance, and consistency with the project glossary.
