# Code Review Checklist

- **Style**: Adheres to the Biome configuration for formatting and linting.

- **Security**: No hardcoded API keys, no improper use of `eval()`, and proper input validation.

- **Best Practices**: Dry code, appropriate naming conventions, and clear documentation.

- **Paths**: Correct quoting for paths with spaces.

- **Environment**: Environment variables accessed via standard APIs (e.g., `os.environ`).
