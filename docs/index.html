Hire Your Own AI Development Team in One Command

An interactive CLI that sets up a fully autonomous team of Claude-powered agents. Pick a team, configure options, and let agents architect, code, review, and test your project.

Get Started View Source

How It Works

Everything starts with an interactive wizard. No flags to memorize — just answer the prompts.

1 Install

curl -fsSL https://raw.githubusercontent.com/Mitriyweb/agent-team/main/install.sh | bash

2 Initialize your project

$ agent-team init

▶ Select a team
  ● software development — team-lead → architect → developer → reviewer → qa
  ○ frontend — team-lead → ui-architect → frontend-dev → design-reviewer → qa
  ○ fullstack — team-lead → architect → fe-dev + be-dev → reviewer → qa
  ○ localization — team-lead → tech-writer → localizer → seo → qa

▶ Planner
  ● Built-in — ROADMAP.md → tasks/plan.md
  ○ OpenSpec — structured proposals (requires @fission-ai/openspec)

▶ Enable human review checkpoints? Yes / No

▶ Enable Obsidian vault for RAG? Yes / No

▶ External review agent (optional)
  ● None
  ○ Codex — OpenAI Codex CLI
  ○ Claude Code — Claude Code CLI
  ○ Gemini CLI — Google Gemini CLI
  ○ Devin / Aider

▶ Enable Telegram notifications? Yes / No

▶ Setup commands to run before cycle (optional)
  nvm use 20, pyenv local 3.10

▶ Enable sound notifications (review/done/failed)? Yes / No

✓ Updated .gitignore
✓ Initialized team: software development
✓ Updated CLAUDE.md with agent-team context
✓ Done. Run agent-team run --plan --all to start.

3 Write a roadmap and go

agent-team plan            # decompose ROADMAP.md into tasks
agent-team run --all       # agents execute every task autonomously

Re-running agent-team init on an existing project pre-selects your current config — just press Enter to keep everything or change what you need.

Why Agent Team?

Interactive Setup

Guided wizard walks you through team, planner, review, and notification options. No docs to read before you start.

Autonomous Execution

Agents work through your roadmap end-to-end — architecture, coding, review, and testing — with zero hand-holding.

Human-in-the-Loop

Optional review checkpoints with audio + Telegram alerts. Approve or reject before agents proceed.

Voice Notifications

Spoken cues on real loop exits — all tasks done, or stopped due to an error in the current run. Review cue fires only in single-task runs, so --all loops don't spam you between tasks. Toggle with --no-sound at init, or drop your own .m4a files into ~/.agent-team/assets/.

Environment Setup

Configure custom commands like nvm use or pyenv local to run before the cycle. Environment variables are captured and propagated to agents.

External Review

Plug in Codex, Gemini, Claude, Devin, or Aider as an independent reviewer. Runs twice: on the generated plan and after each completed task.

Framework Aware

Agents detect your tech stack, lint rules, test runner, and build system dynamically. No hardcoded tool names.

Full Audit Trail

Every tool call is logged. Task reports, cost tracking, and structured audit in .claude-loop/.

Installation

Supports macOS (Apple Silicon & Intel) and Linux (x64 & arm64).

One-line install (Recommended)

Detects your platform, downloads the correct binary, and installs to ~/.local/bin/.

curl -fsSL https://raw.githubusercontent.com/Mitriyweb/agent-team/main/install.sh | bash

npx (no install)

npx @mitriyweb/agent-team init

From source

git clone https://github.com/Mitriyweb/agent-team.git
cd agent-team && bun install && bun run build
ln -sf $(pwd)/dist/agent-team ~/.local/bin/agent-team

Ready-to-use Teams

Select a team during agent-team init. Each has its own agent roles, communication protocol, and quality gates.

Software Development

Team Lead, Architect, Developer, Reviewer, QA. Proof-Loop methodology with frozen spec, developer evidence, and independent QA verdict — all stored per-task in .claude-loop/reports/.

Frontend

UI Architect, Frontend Dev, Design Reviewer, QA. WCAG 2.1 AA compliance, visual regression, performance audits.

Fullstack

Architect, Frontend Dev, Backend Dev, Reviewer, QA. Team lead routes tasks based on scope — frontend, backend, or both.

Localization

Technical Writer, Localizer, SEO Specialist, QA. Content creation and translation pipeline.

Need something different? Create a custom team interactively:

agent-team new-team

Configuration

All options are set interactively during init and saved to agent-team.json. Change anything later with:

agent-team reconfigure

Planners

Built-in: Write ROADMAP.md, run agent-team plan.
OpenSpec: Structured proposals with design docs and task checklists.

External Review

An independent CLI agent reviews both the plan (before execution) and each completed task. Choose from Codex, Claude, Gemini, Devin, or Aider.

Telegram

Real-time notifications on task started, done, failed, and review needed. Set up bot token and chat ID during init.

Obsidian Vault

Connect a vault for RAG integration. Agent knowledge syncs to interlinked Obsidian notes with Dataview metadata.

Setup Commands

Run environment setup (Python, Node versions) or install dependencies automatically before agents start their cycle.

For CI/scripting, every option has a flag equivalent:

agent-team init --team fullstack --planner openspec --external-review codex \
  --telegram-token T --telegram-chat C --setup-commands "nvm use 20" \
  --no-human-review --no-sound

Execution

SDK Mode (default)

Tasks run via @anthropic-ai/claude-agent-sdk — programmatic, in-process, with streaming messages and safety hooks.

agent-team run --all

CLI Mode

Falls back to spawning claude as a subprocess. Useful for debugging or when the SDK is unavailable.

agent-team run --all --cli

Monitoring & Audit

Every task produces logs, reports, and a structured audit trail.

Task Reports

.claude-loop/reports/task-{id}.md — what was done, who did what, test results.

Audit Trail

.claude-loop/audit/audit.jsonl — every tool call with role, agent, tool name, and phase.

Cost Tracking

.claude-loop/logs/ — session ID, model, token usage, and cost per task.

agent-team audit    # Show audit report
